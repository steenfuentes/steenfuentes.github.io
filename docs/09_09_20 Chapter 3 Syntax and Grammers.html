What is Syntax Analysis?

  • Occurs after the scanning of each line of code, terminated by ';', which results in a series of tokens to loaded in for syntactical analysis or parsing.
  • The goal of parsing out tokens is to intepret those tokens and recover the structure described by that series of tokens.
    • If the tokens do not properly enocde a structure, an error is to be reported.

Syntax vs Semantics

  • Syntax refers to the ordering of the typed languaged, the words and symbols.
    • the form or struture of the expressions, statements, and program units
  • Semantics is how the program works. What the program means.
  • - the meaning of expressions, statements, and program units.
  • Together, syntax and semantics provide the language's definition.
  • Compile time errors are said to be a _static issue

Backus-Naur Form (BNF)

  • notation for describing language grammar rules.
  • The notation requires:
    • Terminals: legal strings
    • Nonterminals: concepts of the language that can be further refined
    • Rules: expanding nonterminals to a series of nonterminals and terminals
    • Start: a nonterminal that begins the expansion

The General Problem of Describing Syntax: Terminology

  • A sentence is a string of characers over some alphabet
  • A language is a set of sentences.
  • A lexeme is the lowest level syntactic unit (e.g., *, sum, begin)
  • A token is a category of lexemes (e.g., identifier)

BNF amd Context-Free Grammars

  • Context-Free Grammars

BNF Fundamentals

  • Grammar is a finite set of rules
  • Specific start rules
  • An abstaction (or nonterminal symbol) can have more than one RHS
  • Synactic lists are described using recursion <ident_list> -> ident | ident, <ident_list>
  • A derivation is a repeated application of rules, starting with the start symbol
  • a Leftmost derivation is one in which the leftmost nonterminal in each sentential form is the one that is expanded.
  • a Parse Tree is a hierarchical representation of a derivation
